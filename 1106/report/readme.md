# Assembly 수업 요약 (1106)

## 부울연산 및 비교 명령어
어셈블리에서 조건 처리를 위해 비트 연산과 비교 명령어는 핵심적인 역할을 한다.

| 명령어 | 역할 및 특징 |
| ------ | ----------- |
| AND    | 두 오퍼랜드 비트 AND 연산, 특정 비트 클리어, 오버플로우/캐리 플래그 클리어, 결과에 따라 Sign/Zero/Parity 플래그 수정 |
| OR     | 두 오퍼랜드 비트 OR 연산, 캐리/오버플로우 플래그 클리어, 결과에 따라 Sign/Zero/Parity 플래그 수정 |
| XOR    | 두 오퍼랜드 비트 XOR 연산, 캐리/오버플로우 플래그 클리어, 결과에 따라 Sign/Zero/Parity 플래그 수정 |
| NOT    | 비트 반전, 플래그 비변경 |
| TEST   | 두 오퍼랜드 비트 AND 연산 후 플래그만 수정(결과 저장 안 함), 오버플로우/캐리 플래그 클리어 |
| CMP    | 두 오퍼랜드 비교(뺄셈 연산) 후 플래그 설정, 오퍼랜드 본문은 변경하지 않음 |

이들 명령어를 통해 비트 단위 조작, 집합 연산, 값 비교 및 플래그 설정이 가능하여 조건 점프를 위한 기반을 마련한다.

## 조건 분기 및 점프 명령어
어셈블리에서는 고급 프로그래밍 언어처럼 if, else 문이 없으나, 비교와 조건 점프를 활용해 동일한 제어 흐름을 만든다.

- 조건 점프 명령어 예시:

| 명령어 | 의미 |
| ------ | ------ |
| JE / JZ | Zero 플래그가 1일 때 점프 (같음) |
| JNE / JNZ | Zero 플래그가 0일 때 점프 (다름) |
| JL / JNGE | 부호 있는 작음일 때 점프 |
| JG / JNLE | 부호 있는 큼일 때 점프 |

- 활용 예시: 배열 내 0이 아닌 첫 번째 값을 찾는 루프를 구현 시 조건 점프로 분기 제어

## 암호화 예제
- XOR 명령으로 각 문자 바이트를 암호화 키와 배타적 OR 하여 암호화 및 복호화 수행
- 암호화한 문자열을 다시 같은 키로 XOR 하면 원문이 복구됨

## 조건 루프 명령어
- `LOOPZ`: ECX가 0이 아니고 Zero 플래그가 1이면 루프 계속
- `LOOPNZ`: ECX가 0이 아니고 Zero 플래그가 0이면 루프 계속
- 조건과 루프 카운터 병합 제어가 가능

## 조건 구조화
- 어셈블리에서 while, if-else 등의 구조를 점프 명령으로 작성
- 테이블 기반 분기로 다중 분기 효율화 가능(Procedure Offsets 테이블 이용)

## 고급 조건 흐름 지시자 (MASM 32비트 모드)
- `.IF`, `.ELSE`, `.ELSEIF`, `.ENDIF` 등의 지시문으로 복잡한 분기문 간단하게 작성 가능
- 단, 64비트 모드 MASM에서는 사용 불가

## 어셈블리 조건 처리 요약 표

| 범주           | 주요 명령어 및 개념                          | 역할 및 설명                                    |
| -------------- | ------------------------------------------- | --------------------------------------------- |
| 비트 연산      | AND, OR, XOR, NOT, TEST                     | 조건 검사와 변수 변형, 플래그 상태 설정       |
| 비교           | CMP                                         | 두 수 비교, 플래그 상태 반영                   |
| 조건 점프      | JE, JNE, JL, JG 등                         | 플래그 상태에 따른 프로그램 흐름 제어         |
| 조건 루프      | LOOPZ, LOOPNZ                               | 플래그와 카운터 상태에 따른 반복문 제어       |
| 고급 구조화    | 테이블 기반 분기, 조건 흐름 지시자           | 다중 분기 처리 및 고수준 조건문 단순화         |
| 암호화 활용    | XOR                                        | 간단한 대칭 암호화 및 복호화                   |

## 결론
어셈블리 언어는 저수준 언어임에도 결정 구조 및 반복 제어를 형성하는 다양한 방법을 제공한다. 비트 연산과 비교 명령어로 플래그를 상태를 조작하고, 조건 점프/루프를 통해 제어 흐름을 조절하며, MASM에서 고급 조건 지시자를 활용해 복잡한 조건문도 간결하게 코딩 가능하다. 이러한 기법을 바탕으로 프로그래머는 어셈블리어로도 효율적이고 명확한 조건 처리 로직을 구현할 수 있다.
